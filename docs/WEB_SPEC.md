## 1. 背景与目标

### 1.1 背景

GFEP 运行中产生日志文件；在受控网络环境下，需要为运维提供 **可选的简易 HTTP 服务**，用于在浏览器中 **列举并下载日志**，无需额外工具。

### 1.2 目标

通过配置文件 **按需启用** Web 服务，仅开放 **日志根目录** 下的列举与下载，默认 **关闭** 以降低暴露面。

### 1.3 非目标（本期或另立 SPEC）

- 通用文件管理、上传、在线编辑。
- 日志全文检索、复杂 RBAC、多用户体系（除非另立 SPEC）。

## 2. 功能需求

### 2.1 配置（FR-WEB-001）

| 配置项 | 说明 |
|--------|------|
| `web.enabled` | 默认 `false` |
| `web.listen` | 监听地址与端口 |
| `web.tls.*` | 可选，HTTPS |
| `web.log_root` | 允许列举与下载的根目录（绝对路径或相对工作目录，实现定稿） |
| `web.auth.*` | 至少一种：**HTTP Basic**、**静态 Token**（Header 或 Query）、**mTLS** 之一 |

### 2.2 功能（FR-WEB-002）

- 用户通过浏览器访问后，**必须** 能 **列出** `web.log_root` 下可下载的日志文件（或按日期/滚动规则展示）。
- **必须** 支持 **HTTP 下载**，响应头使用 `Content-Disposition: attachment`，文件名与磁盘一致或按规则映射。
- **必须** 将访问限制在 `web.log_root` 内：解析路径后 **禁止** `../` 等路径穿越；非法路径返回 **404** 或 **403**（实现选一并文档化）。

### 2.3 非功能（FR-WEB-003）

- 大文件下载 **宜** **流式** 输出，避免整文件读入内存。
- **宜** 将监听绑定 **内网或回环**，并在运维文档中提示勿对公网裸奔开启。

---

## 3. 配置项汇总（Web）

```text
web.enabled
web.listen
web.log_root
web.auth.*
web.tls.*                  # 可选
```

---

## 4. 非功能需求（Web）

| ID | 要求 |
|----|------|
| NFR-WEB-001 | 关闭 `web.enabled` 时 **不得** 监听 Web 端口 |
| NFR-WEB-002 | Web 访问与错误 **宜** 可审计（访问日志或应用日志） |
| NFR-WEB-003 | 未启用 Web 时，不改变现有进程其它行为 |

---

## 5. 验收要点（Web）

1. `web.enabled=false` 时无 Web 监听。
2. 开启后：未授权请求无法列出或下载；授权用户可列出并下载 `log_root` 内文件。
3. 使用 `../` 或绝对路径等方式无法访问 `log_root` 之外的文件。

## 7. 与通用 Web 安全实践对照

以下对照常见 Web 服务安全实践，**不表示本期必须全部实现**，可作为增强 backlog。

| 建议项 | 说明 |
|--------|------|
| **HTTP 方法与路由** | 明确仅允许 **GET**（及必要时 HEAD）；禁止调试接口未授权暴露 |
| **目录列表样式** | 是否采用极简 HTML 列表或 JSON API，避免返回服务器版本等敏感头（可按需配置） |
| **CORS** | 若仅内网脚本访问，可默认 **禁用 CORS**；若跨域，显式白名单 |
| **速率限制** | 对下载与鉴权失败 **限流**，防拖库与暴力猜 Token |
| **日志轮转** | 列举时是否排除「当前正在写入」的文件、文件名 **排序规则**（时间倒序等） |
| **TLS** | 生产环境 **宜** 仅 HTTPS；证书与密码 **不可** 写死在仓库 |

**MQTT 侧平台对照** 见 [MQTT_SPEC.md](./MQTT_SPEC.md) 第 12 节。
