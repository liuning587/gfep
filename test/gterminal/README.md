# gterminal — 698 模拟终端

用于联调 **gfep** 的简易 **DL/T 698.45** 终端侧客户端：建立 TCP 连接、发送登录帧；在收到指定**抄读电压**（单播地址）或**广播读时钟**（`AA` 广播地址）下行报文时，按本终端**实际通信地址**组应答并重算 **HCS/FCS**（与 `gfep/zptl` 一致）。

## 环境

- 与主工程相同：**Go 1.21+**
- 在仓库**根目录**执行 `go run` / `go build`（模块路径为 `gfep/test/gterminal`）

## 快速运行

1. 启动 gfep，并确认 **698 端口**与下面 `-server` 一致（默认 `conf/gfep.json` 中 `TCPPort` 常为 `20083`）。
2. 在仓库根目录执行：

```bash
go run ./test/gterminal
```

默认：连接 `127.0.0.1:20083`，通信地址 `270300000026`（12 位十六进制），单实例。

主站经 FEP 下发：

1. **抄读电压**（主站→终端，单播 `0x05` + 6 字节地址与 `-addr` 一致）→ 终端回电压应答（模板与现场样例一致，地址域为本机地址）。  
2. **广播读时钟**（主站→终端，地址域 `0x4F` 后一段 **`0xAA` 填充**，再跟读时钟 APDU；控制字 `0x43`）→ 各在线终端回**读时钟应答**（`L=0x21`，`0xC3`，地址为 **`0x05` + 本机 6 字节**）。

说明：现场常见两种形态，本模拟器**均识别**（**不校验 HCS/FCS**，因不少工具与 `L` 域、CRC 不一致）：

- **规范型**：`… 4F` + n×`AA` + `52` + 2 字节 HCS + 8 字节 `DA 7D 05 01 01 40 00 02` + FCS + `16`（n 常为 16，总长 35）。  
- **工具/抓包型**：`… 4F` + n×`AA`（如 **24** 字节）+ 连续 11 字节 `52 DA 7D 05 01 01 40 00 02 00 00` + FCS + `16`（总长可能 **43**，与 `L=0x21` 不符且 `zptl` 校验失败属正常）。

能通过 `Ptl698_45CompleteFrameLen` 的帧走严格拆帧；否则按上述结构做宽松定界。

## 命令行参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `-server` | `127.0.0.1:20083` | FEP 监听地址 `host:port` |
| `-addr` | `270300000026` | **6 字节通信地址**，须为 **12 个十六进制字符**（无空格） |
| `-n` | `1` | 并发模拟的终端连接数（每个连接一个 goroutine） |
| `-unique` | `false` | 为 `true` 时，在基础地址上按实例序号改写**最后 2 字节**（`cNo` 高/低字节），避免大规模模拟时地址冲突 |
| `-v` | `false` | 打印登录、收帧、电压/广播时钟应答等十六进制调试输出 |

### 示例

单终端、指定地址、详细日志：

```bash
go run ./test/gterminal -addr 270300000026 -n 1 -v
```

多终端压测（地址尾字节随序号递增，需 FEP 与主站侧路由支持）：

```bash
go run ./test/gterminal -server 192.168.1.10:20083 -n 5000 -unique
```

编译为可执行文件（在仓库根目录）：

```bash
go build -o gterminal.exe ./test/gterminal
./gterminal.exe -addr 270300000026 -v
```

## 行为说明

- **登录**：上电后发送一帧与历史联调一致的 32 字节登录报文；通信地址写入与下行抄读帧相同的地址域（`0x05` + 6 字节）。
- **拆帧**：使用 `gfep/zptl` 的 `Ptl698_45CompleteFrameLen` 处理 TCP 粘包/半包。
- **抄读电压**：主站→终端、`0x05` + 当前实例 6 字节地址、且 `[11:]` 与内置电压读 APDU 前缀一致 → 回电压应答模板，**仅替换 `[5:11]` 地址**并重算 **HCS/FCS**。  
- **广播读时钟**：在 `0x68 21 00 43 4F` 后跳过连续 `0xAA`，再匹配「规范 52+HCS+8 用户数据」或「11 字节 `52 DA 7D…00 00`」任一种直至 FCS+`16` → 回读时钟应答并重算 CRC。  
- **并发写**：每个连接单 goroutine 读循环内写应答，避免与另一路发送并发 `Write` 同一 TCP 连接。

## 与 FEP 配置的关系

- 模拟终端必须连到 gfep 对外监听的 **698 业务口**（与主站工具相同端口或按你现场拆分端口为准）。
- `-addr` 须与主站抄读帧中的**终端通信地址**一致，否则 FEP 无法把下行转到该连接。
- 大规模 `-n` 时请关注操作系统 **fd 上限**、gfep 的 `MaxConn` / `WorkerPoolSize` / 转发队列等配置。
